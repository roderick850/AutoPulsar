🚀 Inicio rápido

En 3 pasos

  1. Agregá tus scripts — clic en + Agregar y seleccioná los .exe de TinyTask.
  2. Configurá repeticiones y pausas — doble clic en cada columna para editar.
  3. Ejecutá — clic en ▶ Iniciar o presioná F10.

🔄 Modos de loop

ModoComportamientoEjemplo
onceEjecuta la lista 1 sola vezAutomatización puntual
fixedRepite N veces con pausa entre loopsFarmear 10 veces con pausa de 5s
infiniteRepite sin fin hasta que lo detengasAFK farming continuo

⚙️ Configuración

Modo: once | fixed | infinite

× repeticiones: Solo aplica en modo fixed

Pausa (s): Tiempo de espera entre cada loop completo

Total: Tiempo estimado total de ejecución

💡 Tip: Con infinite, el script con ☑️ 1° solo se ejecuta en la primera vuelta. Ideal para scripts de preparación.

🎯 Condiciones visuales

Las condiciones permiten que un script solo se ejecute si ciertos iconos están (o no están) visibles en pantalla. Esto evita que el automatismo falle cuando el juego cambia de pantalla inesperadamente.

Tipos de condición

TipoIconoSignificado
Requerir✅El icono DEBE estar visible para ejecutar
Bloquear❌El icono NO debe estar visible para ejecutar

Modos de combinación

ModoLógicaCuándo usarlo
ANDTODAS las condiciones deben cumplirseVarias verificaciones necesarias a la vez
ORAL MENOS UNA debe cumplirseVarias pantallas válidas alternativas

Acciones

AcciónComportamiento
requireSolo ejecuta si las condiciones se cumplen. Si no, salta el script.
repeat_untilEjecuta el script repetidamente hasta que las condiciones se cumplan (o se alcance el máximo de intentos).

🔄 Repeat Until

Ideal para acciones tipo "clickear hasta que aparezca X":

  • Detener al: Encontrar el icono (match) o cuando desaparezca (no_match)
  • Máx. intentos: Límite de repeticiones (0 = sin límite)
  • Intervalo: Segundos entre cada verificación

📊 Multi-muestreo

Para imágenes difíciles (juegos con renderizado variable), configurá Muestreo:

  • Verificar N veces: Toma múltiples capturas rápidas (ej: 5)
  • Aceptar si ≥ X%: Porcentaje mínimo de aciertos (ej: 60% = 3 de 5)
💡 Tip bloqueo inverso: Usá 4+ condiciones tipo ❌ Bloquear con modo AND para ejecutar un script solo cuando ninguna de esas pantallas está visible — ideal para pantallas de "else" o fallback.

⏳ Reintentos

Si las condiciones no se cumplen, podés configurar reintentos automáticos cada N segundos.

🆘 Script de recuperación

Si un script falla X veces seguidas, ejecutá un script alternativo (ej: reiniciar el juego, volver al menú).

🔍 Diagnóstico de matching

El botón 🔍 Probar ejecuta 5 capturas rápidas y te muestra:

Resultado del diagnóstico

  • Tasa de acierto: X/5 pruebas exitosas
  • Min / Max / Avg diff: Rango de diferencia real
  • Variación: Qué tanto cambia entre frames
  • Tolerancia SEGURA: Valor recomendado (max diff + 15%)
  • Clasificación: ✅ CONFIABLE (≥80%) | ⚠️ INESTABLE | ❌ NO CONFIABLE

🧪 Probar TODAS

El botón 🧪 Todas evalúa todas las condiciones juntas con su lógica (AND/OR, require/block) y te muestra exactamente cuáles pasan, cuáles fallan, y si el script se ejecutaría o no.

💡 Regla de oro: Usá la tolerancia SEGURA que sugiere el diagnóstico, no la mínima. El juego renderiza distinto cada frame y necesitás cubrir el peor caso.

Hotkey global

Por defecto F10 inicia/detiene la ejecución incluso con el juego en primer plano. Podés cambiarla desde el desplegable.

Barra de progreso

Muestra el progreso global: scripts ejecutados / total. En modo infinite muestra el progreso por loop.

Mini Bar

Activala desde Ver → Mini Bar. Es una barra flotante y compacta que muestra:

  • Script actual y progreso
  • Tiempo restante estimado
  • Barra de progreso

Ideal para gaming en un solo monitor: se superpone al juego sin estorbar.

⚠️ Importante: Las duraciones deben ser realistas. Si un script tarda 30s en ejecutarse, poné Dur: 30. Un valor muy bajo causa que el siguiente script se ejecute antes de tiempo. Un valor muy alto desperdicia tiempo.
💡 Flujo recomendado para juegos:
  1. Creá un perfil por juego
  2. Agrupá scripts por "fase" (compra, farm, venta)
  3. Configurá condiciones visuales en los scripts clave
  4. Usá repeat_until para acciones repetitivas con condición de salida
  5. Agregá un script fallback con ❌ Bloquear para pantallas inesperadas
  6. Probá con ▶1 Seleccionado antes de lanzar el loop completo
