🎬 Macros nativas

Las macros son el corazón de AutoPulsar. Te permiten grabar, editar y reproducir secuencias de teclado y ratón sin depender de TinyTask ni ninguna herramienta externa.

Crear una macro

  1. Clic en 🎬 Macro
  2. Asigna un nombre a la macro
  3. Opción A — Grabar: clic en ⏺️ Grabar, realiza las acciones, clic en ⏹️ Detener
  4. Opción B — Manual: usa los botones ➕ Añadir Tecla, ➕ Añadir Click y ➕ Añadir Espera
  5. Revisa el timeline, edita lo necesario
  6. Prueba con ▶️ Probar
  7. Clic en 💾 Guardar Macro y Cerrar

Cada zona numerada se explica en el panel derecho de la imagen.

Timeline visual

Cada acción se muestra como una tarjeta en el timeline:

  • Teclas — keycap coloreado por tipo (modificador, función, flecha, etc.)
  • Clics — indicador de botón (izquierdo/derecho/medio) con coordenadas
  • Esperas — pausa configurable en segundos
  • Sub-filas de espera — muestra el tiempo hasta la siguiente acción
  • Tiempo acumulado — cuánto tarda la macro desde el inicio

🔄 Modos de loop

ModoComportamientoEjemplo
onceEjecuta la lista 1 sola vezAutomatización puntual
fixedRepite N veces con pausa entre loopsFarmear 10 veces con pausa de 5s
infiniteRepite sin fin hasta que lo detengasAFK farming continuo
first_matchEjecuta solo el primer item con condición cumplida, luego re-evalúaAutomatización por estados de pantalla

⚙️ Configuración

Modo: once | fixed | infinite | first_match

× repeticiones: Solo aplica en modo fixed

Pausa (s): Tiempo de espera entre cada loop completo

Total: Tiempo estimado total de ejecución

💡 Tip: Con infinite, el item con ☑️ 1° solo se ejecuta en la primera vuelta. Ideal para acciones de preparación.

🎯 Condiciones visuales

Las condiciones permiten que un script solo se ejecute si ciertos iconos están (o no están) visibles en pantalla. Esto evita que el automatismo falle cuando el juego cambia de pantalla inesperadamente.

Tipos de condición

TipoIconoSignificado
Requerir✅El icono DEBE estar visible para ejecutar
Bloquear❌El icono NO debe estar visible para ejecutar

Modos de combinación

ModoLógicaCuándo usarlo
ANDTODAS las condiciones deben cumplirseVarias verificaciones necesarias a la vez
ORAL MENOS UNA debe cumplirseVarias pantallas válidas alternativas

Acciones

AcciónComportamiento
requireSolo ejecuta si las condiciones se cumplen. Si no, salta el script.
repeat_untilEjecuta el script repetidamente hasta que las condiciones se cumplan (o se alcance el máximo de intentos).

🔄 Repeat Until

Ideal para acciones tipo "clickear hasta que aparezca X":

  • Detener al: Encontrar el icono (match) o cuando desaparezca (no_match)
  • Máx. intentos: Límite de repeticiones (0 = sin límite)
  • Intervalo: Segundos entre cada verificación

📊 Multi-muestreo

Para imágenes difíciles (juegos con renderizado variable), configura Muestreo:

  • Verificar N veces: Toma múltiples capturas rápidas (ej: 5)
  • Aceptar si ≥ X%: Porcentaje mínimo de aciertos (ej: 60% = 3 de 5)
💡 Tip bloqueo inverso: Usa 4+ condiciones tipo ❌ Bloquear con modo AND para ejecutar un script solo cuando ninguna de esas pantallas está visible — ideal para pantallas de "else" o fallback.

⏳ Reintentos

Si las condiciones no se cumplen, puedes configurar reintentos automáticos cada N segundos.

🆘 Script de recuperación

Si un script falla X veces seguidas, ejecuta un script alternativo o una macro de la playlist (ej: reiniciar el juego, volver al menú).

  • Externo (.exe): lanza un ejecutable externo con subprocess.
  • Macro: busca la macro por nombre en tu playlist y la reproduce con el motor de macros (clicks, teclas, delays).

🔍 Diagnóstico de matching

El botón 🔍 Probar ejecuta 5 capturas rápidas y te muestra:

Resultado del diagnóstico

  • Tasa de acierto: X/5 pruebas exitosas
  • Min / Max / Avg diff: Rango de diferencia real
  • Variación: Qué tanto cambia entre frames
  • Tolerancia SEGURA: Valor recomendado (max diff + 15%)
  • Clasificación: ✅ CONFIABLE (≥80%) | ⚠️ INESTABLE | ❌ NO CONFIABLE

🧪 Probar TODAS

El botón 🧪 Todas evalúa todas las condiciones juntas con su lógica (AND/OR, require/block) y te muestra exactamente cuáles pasan, cuáles fallan, y si el script se ejecutaría o no.

💡 Regla de oro: Usa la tolerancia SEGURA que sugiere el diagnóstico, no la mínima. El juego renderiza distinto cada frame y necesitas cubrir el peor caso.

Mini Bar

Actívala desde Ver → Mini Bar. Es una barra flotante y compacta que muestra:

  • Item actual y progreso
  • Tiempo restante estimado
  • Barra de progreso

Ideal para gaming en un solo monitor: se superpone al juego sin estorbar.

⚠️ Importante: Las duraciones deben ser realistas. Si un script tarda 30s en ejecutarse, pon Dur: 30. Un valor muy bajo causa que el siguiente script se ejecute antes de tiempo. Un valor muy alto desperdicia tiempo.
💡 Flujo recomendado para juegos:
  1. Crea un perfil por juego
  2. Graba macros nativas para las acciones repetitivas
  3. Agrupa items por "fase" (compra, farm, venta)
  4. Configura condiciones visuales en los items clave
  5. Usa repeat_until para acciones repetitivas con condición de salida
  6. Agrega un script fallback con ❌ Bloquear para pantallas inesperadas
  7. Prueba con ▶1 Seleccionado antes de lanzar el loop completo
